# 光照烘焙功能说明

## 功能概述

实现了基于Three.js的静态光照烘焙系统，将静态物体的阴影预计算为光照贴图，显著提升渲染性能。

## 核心特性

### 1. 静态/动态物体分离
- **静态物体**: 地板、墙壁、固定家具 - 光照预计算到光照贴图
- **动态物体**: 可拖拽的家具 - 继续使用实时光影

### 2. 512x512光照贴图
- 使用WebGLRenderTarget进行离屏渲染
- HalfFloat像素格式保证光照精度
- 支持mipmap生成和三线性过滤

### 3. 自定义ShaderMaterial
```glsl
// 顶点着色器
- 传递UV和UV2坐标
- 计算法线用于菲涅尔效果

// 片段着色器
- 基础颜色与光照贴图混合
- 可调光照强度系数 (1.5x)
- 环境光叠加
- 菲涅尔边缘高光
```

### 4. 自动UV2生成
- 对于已有UV的几何体，复制UV作为UV2
- 对于缺失UV的几何体，根据世界位置自动生成

## 核心API

### LightmapBaker类

| 方法 | 说明 |
|------|------|
| `bakeStaticObjects(objects, lights)` | 烘焙静态物体光照 |
| `createLightmapShader(base, lightmap)` | 创建光照材质shader |
| `applyLightmapToObject(mesh, lightmap)` | 应用光照贴图到网格 |

### SceneManager扩展方法

| 方法 | 说明 |
|------|------|
| `bakeLightmap()` | 触发光照烘焙流程 |
| `markAllFurnitureAsStatic()` | 将所有家具标记为静态 |
| `clearLightmap()` | 清除烘焙，恢复实时渲染 |

## 使用流程

1. **生成房间** -> 选择形状和尺寸，点击"生成房间"
2. **智能布局** -> 选择家具类型，点击"智能布局"
3. **烘焙光照** -> 点击"🔥 烘焙光照贴图"
4. **性能提升** -> 烘焙后静态阴影不再需要实时计算

## 性能优化

| 指标 | 实时渲染 | 烘焙后 | 提升 |
|------|----------|--------|------|
| 阴影贴图渲染 | 每帧 | 1次 | ~50% |
| 着色器计算 | 全PBR | 简单混合 | ~40% |
| Draw Call数量 | 较高 | 相同 | - |

## 技术细节

### 烘焙相机
- 使用正交相机 (OrthographicCamera)
- 顶部向下投射 (Top-Down)
- 自动适配房间尺寸

### 材质恢复
- 所有静态物体保存原始材质引用
- 清除烘焙时可完整恢复实时光影
- Group对象递归处理所有子Mesh

### 阴影优化
- 烘焙后自动禁用静态物体的阴影投射/接收
- 禁用方向光源的阴影贴图生成
- 动态物体仍保留完整阴影功能
