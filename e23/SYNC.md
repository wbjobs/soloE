# Markdown Notes - 同步功能说明

## 概述

本应用实现了基于 Yjs CRDT（Conflict-free Replicated Data Type）的实时同步功能，支持多客户端离线编辑并自动解决冲突。

## 技术架构

### 1. 同步服务器 (Node.js)

**位置**: `server/src/server.js`

**功能**:
- WebSocket 服务器 (端口 1234)
- 管理所有 Yjs 文档
- LevelDB 持久化存储
- 客户端连接管理
- 广播更新到所有连接的客户端

**依赖**:
- `ws` - WebSocket 库
- `yjs` - CRDT 库
- `level` - 持久化存储
- `uuid` - 客户端唯一标识

### 2. 客户端同步服务

**位置**: `src/sync.ts`

**功能**:
- WebSocket 连接管理
- 自动重连机制
- 离线更新缓存
- Yjs 文档管理
- 网络状态回调

### 3. 编辑器集成

**位置**: `src/components/MarkdownEditor.tsx`

**功能**:
- Yjs 文档与输入框双向绑定
- 差异计算与增量更新
- 实时同步状态显示

## CRDT 冲突解决机制

### Yjs 工作原理

Yjs 使用基于状态的 CRDT 算法：

1. **操作转换**: 每个编辑操作被编码为一个小的二进制更新
2. **向量时钟**: 跟踪每个客户端的操作顺序
3. **自动合并**: 多个客户端的更新自动合并，无需冲突解决
4. **最终一致性**: 所有客户端最终达到相同状态

### 文本类型 (Y.Text)

Y.Text 是一个可分割的序列，支持：
- `insert(pos, text)` - 插入文本
- `delete(pos, length)` - 删除文本
- `observe(callback)` - 监听变化

### 并发编辑示例

```
客户端 A: 在位置 0 插入 "Hello"
客户端 B: 在位置 0 插入 "World"

结果: "WorldHello" 或 "HelloWorld" (取决于时间戳)
但所有客户端会得到相同结果
```


```bash

```

### 3. 多客户端测试

打开多个 Tauri 应用窗口，编辑同一篇笔记：

- 窗口 A: 编辑笔记 #1
- 窗口 B: 同时编辑笔记 #1
- 结果: 两个窗口的内容自动同步，冲突被自动解决

## 离线-在线工作流

### 离线模式

1. 同步服务器未运行或网络断开
2. 应用显示"离线"状态
3. 用户继续编辑笔记
4. 所有编辑操作被缓存到本地 pendingUpdates 队列
5. 笔记正常保存到本地 SQLite 数据库

### 重新连接

1. 网络恢复或服务器启动
2. SyncService 自动重连（每 3 秒重试）
3. 连接成功后，自动发送所有缓存的更新
4. 从服务器拉取最新状态并合并
5. UI 更新显示"已连接"状态

## 数据流向

```
         │                       │                        │
         ▼                       ▼                        ▼
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│  离线更新缓存   │     │  持久化存储     │     │  离线更新缓存   │

```

## API 消息协议

### 1. Sync 消息 (客户端 → 服务器)

```json
{
  "type": "sync",
  "docId": "1",
  "updates": [
    [0, 255, 13, ...],  // Uint8Array 形式的 Yjs 更新
    [1, 2, 3, ...]
  ]
}
```

### 2. SyncResponse 消息 (服务器 → 客户端)

```json
{
  "type": "sync-response",
  "docId": "1",
  "update": [0, 255, 13, ...]  // 完整文档状态
}
```

### 3. Update 消息 (双向)

```json
{
  "type": "update",
  "docId": "1",
  "update": [0, 255, 13, ...]  // 增量更新
}
```

## 关键特性

✅ **CRDT 冲突解决**: 使用 Yjs 自动处理并发编辑  
✅ **离线优先**: 断网时可正常使用，重连后自动同步  
✅ **增量同步**: 只传输差异，节省带宽  
✅ **自动重连**: 网络断开后自动尝试重连  
✅ **本地持久化**: SQLite + LevelDB 双重存储  
✅ **状态显示**: UI 显示在线/离线状态  
✅ **多客户端支持**: 任意数量客户端同时编辑
