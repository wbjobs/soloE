## 1. 产品概述

WebAssembly稀疏矩阵并行乘法计算平台，利用Rust+WASM技术实现高性能科学计算，通过Web Worker避免UI阻塞，支持大规模稀疏矩阵运算。

- 解决JavaScript在大规模数值计算中的性能瓶颈问题
- 为科研、工程、机器学习领域提供浏览器端的高性能计算工具
- 提供直观的性能对比和计算可视化界面

## 2. 核心功能

### 2.1 用户角色
| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 普通用户 | 无需注册 | 使用所有计算功能、导入导出矩阵、查看计算历史 |

### 2.2 功能模块
1. **矩阵计算页面**: 矩阵输入/生成、计算引擎选择、参数配置、计算执行
2. **性能对比页面**: JS vs WASM性能对比图表、内存占用分析、进度可视化
3. **数据管理页面**: 矩阵导入导出、历史记录查看、矩阵存储管理
4. **结果展示页面**: 计算结果可视化、数据导出、统计分析

### 2.3 页面详情
| 页面名称 | 模块名称 | 功能描述 |
|---------|----------|---------|
| 矩阵计算 | 矩阵输入 | 支持随机生成、手动输入、CSV/Parquet导入 |
| 矩阵计算 | 计算引擎 | JS/WASM切换、线程数配置、CSR格式选择 |
| 矩阵计算 | 实时监控 | 进度条、内存占用、CPU使用率实时显示 |
| 性能对比 | 图表展示 | 执行时间对比柱状图、内存变化折线图 |
| 数据管理 | 导入导出 | CSV/Parquet格式解析、文件上传下载 |
| 数据管理 | 历史记录 | 计算历史列表、结果回溯、重复计算 |

## 3. 核心流程

用户上传或生成稀疏矩阵 → 选择计算引擎(JS/WASM) → 配置计算参数 → 启动计算(Worker后台执行) → 实时查看进度和内存 → 获得计算结果 → 导出数据或保存到历史记录

```mermaid
flowchart LR
    A["矩阵输入"] --> B["选择计算引擎"]
    B --> C{"WASM?"}
    C -->|是| D["Web Worker + WASM"]
    C -->|否| E["Web Worker + JS"]
    D --> F["实时进度更新"]
    E --> F
    F --> G["计算完成"]
    G --> H["结果展示/导出"]
    H --> I["保存历史记录"]
```

## 4. 用户界面设计

### 4.1 设计风格
- **主色调**: 深蓝色 (#165DFF) 代表科技和专业
- **辅助色**: 青色 (#0FC6C2) 用于进度和成功状态，橙色 (#FF7D00) 用于警告
- **按钮风格**: 圆角矩形，微妙阴影，hover时轻微上浮动画
- **字体**: JetBrains Mono (代码/数字) + Inter (正文)
- **布局风格**: 卡片式布局，清晰的视觉层次，数据可视化区域突出
- **图标风格**: 线性图标，统一24px尺寸，科技感强

### 4.2 页面设计概述
| 页面名称 | 模块名称 | UI元素 |
|---------|----------|--------|
| 矩阵计算 | 控制面板 | 深色侧边栏，参数滑块，切换开关 |
| 矩阵计算 | 监控面板 | 实时图表，进度动画，数据卡片 |
| 性能对比 | 图表区域 | 交互式ECharts图表，数据tooltip |
| 数据管理 | 文件操作 | 拖拽上传区域，文件列表，下载按钮 |

### 4.3 响应式
- Desktop-first设计，12列网格布局
- 平板端：侧边栏折叠为图标，图表自适应
- 移动端：单列布局，触控优化，简化功能

### 4.4 动画效果
- 页面加载：元素淡入+上移动画，错开0.1s
- 进度条：平滑增长动画，脉冲效果提示活跃状态
- 卡片hover：轻微scale+阴影加深
- 数据更新：数字滚动动画，数值变化高亮
