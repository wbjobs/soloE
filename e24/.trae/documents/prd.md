# 2D流体模拟产品需求文档 (PRD)

## 1. 项目概述

### 1.1 项目背景
本项目旨在实现一个基于WebGPU的高性能2D流体实时模拟器，在浏览器中展示流体动力学效果。

### 1.2 项目目标
- 实现简化的Navier-Stokes方程求解器
- 提供60fps以上的实时流体模拟效果
- 支持鼠标交互，允许用户"搅动"流体
- 使用WebGPU API实现GPU加速计算

### 1.3 目标用户
- 计算机图形学爱好者
- 物理模拟学习者
- 前端开发者（学习WebGPU）

## 2. 核心功能需求

### 2.1 流体模拟核心
- **对流(Advection)**: 使用半拉格朗日方法进行对流计算
- **扩散(Diffusion)**: 使用雅可比迭代求解扩散项
- **投影(Projection)**: 求解泊松方程使速度场无散度

### 2.2 渲染系统
- 全屏三角形渲染
- 速度场或密度场可视化
- 颜色映射：根据流体属性映射为RGB颜色

### 2.3 交互系统
- 鼠标位置跟踪
- 鼠标移动时向速度场添加力
- 可调节的交互强度和半径

### 2.4 性能指标
- 模拟分辨率：至少256x256网格
- 帧率：60fps以上
- 支持实时参数调整

## 3. 技术栈要求

| 技术 | 版本 | 用途 |
|------|------|------|
| TypeScript | 5.x | 主要开发语言 |
| WebGPU | - | GPU加速计算和渲染 |
| WGSL | - | 着色器语言 |
| Vite | 5.x | 构建工具 |

## 4. 用户界面设计

### 4.1 布局结构
- 主画布：全屏流体模拟展示
- 控制面板（可选）：参数调节
- 简单的使用说明

### 4.2 视觉风格
- 深色背景
- 流体使用渐变色彩（蓝绿色调）
- 高对比度，流体运动清晰可见
- 科技感、现代感的设计风格

### 4.3 交互反馈
- 鼠标悬停时显示交互区域
- 流畅的动画过渡
- 无明显卡顿

## 5. 项目结构

```
e24/
├── src/
│   ├── main.ts           # 入口文件
│   ├── FluidSimulation.ts # 流体模拟核心逻辑
│   ├── Renderer.ts       # WebGPU渲染器
│   └── types.ts          # 类型定义
├── shaders/

│   ├── velocity_update.wgsl # 速度更新着色器
│   └── render.wgsl       # 渲染着色器
├── index.html
├── package.json
├── tsconfig.json
└── vite.config.ts
```

## 6. 验收标准

1. **功能完整性**
   - 流体能够自然流动
   - 支持鼠标交互"搅动"
   - 三个核心算法步骤正确实现

2. **性能要求**
   - 在主流浏览器中运行流畅
   - 帧率稳定在30fps以上

3. **代码质量**
   - TypeScript类型正确
   - 代码结构清晰，有适当注释
   - 遵循WebGPU最佳实践
